# Границы приложения и список экранов (Фаза 0.1)

**Версия:** 1.0  
**Дата:** март 2025  
**Принцип:** вся работа только в телефоне, без своего бэкенда.

---

## 1. Сценарии первой версии (MVP)

### 1.1 Настройки

- Пользователь вводит и сохраняет:
  - **Wildberries API-токен** (из личного кабинета WB, категория «Отзывы и Вопросы»).
  - **Yandex API-ключ** и **folder_id** (каталог в Yandex Cloud).
- Настраивает пороги автоотправки:
  - максимальная сложность для автоответа (например, ≤ 3);
  - минимальная уверенность (например, ≥ 0.8);
  - минимальный рейтинг отзыва для автоответа (например, ≥ 3).
- Редактирует **blacklist** — список слов/фраз, при наличии которых отзыв всегда уходит на модерацию (возврат, брак, юрист, суд и т.п.).
- Все данные хранятся только на устройстве (без отправки на наш сервер).

### 1.2 Загрузка отзывов

- Пользователь запускает загрузку (кнопка «Обновить» или фоновая синхронизация).
- Приложение вызывает **Wildberries API** с телефона, получает список отзывов.
- Отзывы сохраняются в локальную БД (Room). Дубликаты по id не создаём, обновляем при повторной загрузке.

### 1.3 Обработка отзыва (на устройстве)

Для каждого нового/необработанного отзыва:

1. **Классификация** (локально): категория, оценка сложности (1–5), срабатывание blacklist.
2. **RAG (опционально в v1):** поиск похожих отзывов в локальном архиве, формирование контекста для промпта.  
   **Решение для v1:** в первой версии **RAG не делаем** — достаточно классификации + генерации по одному отзыву. RAG добавим в следующей итерации.
3. **Генерация ответа:** запрос к **Yandex GPT API** с телефона (промпт + текст отзыва, при v2 — + контекст RAG).
4. **Решение (локально):** по правилам (пороги, blacklist) — либо **AUTO_SEND** (сразу отправить ответ в WB), либо **MODERATE** (записать в очередь модерации).
5. При AUTO_SEND — приложение отправляет ответ в WB API и помечает отзыв как обработанный в Room.
6. При MODERATE — запись в локальную очередь модерации (та же Room), статус «на модерации».

### 1.4 Модерация

- Экран «Очередь модерации»: список отзывов со статусом «на модерации».
- Для каждого: текст отзыва, рейтинг, артикул, сгенерированный ответ.
- Действия: **Одобрить** (отправить ответ в WB как есть), **Редактировать** (изменить текст и отправить), **Отклонить** (не отправлять, снять с очереди или пометить отклонённым).
- Отправка ответа — вызов **Wildberries API** с телефона (PATCH/POST ответа на отзыв).

### 1.5 Статистика

- Подсчёт по локальной БД (Room): обработано за сегодня/неделю, сколько автоответов, сколько в очереди модерации, сколько отклонено.
- Без бэкенда — только агрегаты по данным на устройстве.

## 2. Решение по RAG для первой версии

- **В первой версии (MVP) RAG не реализуем.**  
  Генерация ответа — только по тексту текущего отзыва + системному промпту.  
  Это упрощает реализацию и ускоряет выход первой версии.
- **Во второй итерации** можно добавить: локальный архив отзывов, эмбеддинги (Yandex Embeddings API или локальная модель), поиск похожих и передачу контекста в Yandex GPT.

## 3. Список экранов (первая версия)

| № | Экран | Описание |
|---|--------|----------|
| 1 | **Старт / Проверка настроек** | При первом запуске или при отсутствии токенов — переход в «Настройки». Иначе — на главный экран (список отзывов или дашборд). |
| 2 | **Настройки** | Поля: WB API-токен, Yandex API-ключ, folder_id. Секция порогов (сложность, уверенность, мин. рейтинг). Редактируемый blacklist (список строк). Кнопка «Сохранить». Секреты хранить в зашифрованном виде. |
| 3 | **Список отзывов** | Список из Room. Фильтр: все / новые (необработанные) / на модерации / обработанные. Кнопка «Обновить» (загрузка из WB). Отображение: краткий текст, рейтинг, статус, дата. Переход в детали по тапу. |
| 4 | **Детали отзыва** | Полный текст отзыва, рейтинг, артикул, автор (если есть), дата. Сгенерированный ответ (если уже сгенерирован). Кнопки: Одобрить / Редактировать / Отклонить (для «на модерации»). Для необработанного — кнопка «Обработать» (запуск пайплайна: классификация → генерация → решение). |
| 5 | **Очередь модерации** | Подмножество списка отзывов со статусом «на модерации». Те же действия: одобрить / редактировать / отклонить. Можно реализовать как фильтр на экране «Список отзывов» или отдельный экран. |
| 6 | **Статистика** | Карточки/числа: обработано за сегодня, за неделю; автоответов; в очереди модерации; отклонено. Данные из Room. |

**Навигация:** нижняя навигация или drawer: Список отзывов | Очередь модерации | Статистика | Настройки.

---

## 4. Итог по границам MVP

- **Входит в v1:** настройки (токены, пороги, blacklist), загрузка отзывов из WB, классификация по правилам на устройстве, генерация ответа через Yandex GPT с телефона, движок решений (авто/модерация), очередь модерации, отправка ответов в WB с телефона, статистика по Room.
- **Не входит в v1:** RAG (поиск похожих отзывов), on-device ML-модель для классификации (только правила), push-уведомления (можно добавить позже).

После согласования этого документа можно переходить к 0.2 (описание API WB и Yandex) и далее к реализации.
