# Yandex Cloud API — вызовы с телефона (Фаза 0.2)

**Назначение:** приложение без бэкенда вызывает Yandex GPT (и при необходимости Yandex Embeddings) с устройства для генерации ответов на отзывы.  
**Официальная документация:** [Yandex Cloud — Yandex GPT](https://cloud.yandex.ru/docs/yandexgpt/), [AI Studio](https://yandex.cloud/ru/docs/ai-studio/).

---

## 1. Аутентификация и каталог

- **API-ключ:** создаётся в Yandex Cloud в нужном каталоге. Пользователь вводит ключ в настройках приложения; хранить только на устройстве в зашифрованном виде.
- **Folder ID (идентификатор каталога):** обязательный параметр для вызовов. Пользователь вводит в настройках (берётся из консоли Yandex Cloud). Передаётся в заголовке запроса, например: `x-folder-id: <folder_id>`.
- **Заголовок авторизации:** обычно `Authorization: Api-Key <api_key>` (точный формат — по актуальной документации Yandex).

---

## 2. Yandex GPT — генерация текста (completion)

### 2.1 Назначение

Получить сгенерированный ответ на отзыв: на вход — системный промпт + текст отзыва (и при v2 — контекст RAG), на выход — текст ответа продавца.

### 2.2 Базовый URL и эндпоинт

- **Базовый URL:** как в документации Yandex (например, `https://llm.api.cloud.yandex.net` или хост для REST completion — уточнять по актуальной документации).
- **Эндпоинт:** обычно что-то вроде `/foundationModels/v1/completion` или `/v1/completion` (точный путь — из официальной документации Yandex GPT Text Generation API).

### 2.3 Формат запроса (типичный для completion)

- **Метод:** `POST`.
- **Заголовки:**
  - `Authorization: Api-Key <api_key>`
  - `x-folder-id: <folder_id>`
  - `Content-Type: application/json`
- **Тело (JSON), концептуально:**
  - Указание модели (например `modelUri: "gpt://.../yandexgpt/latest"` или аналог из документации).
  - `completionOptions`: температура, max_tokens и т.д.
  - `messages`: массив сообщений, например:
    - роль `system` — системный промпт (инструкция: «Ты помощник продавца. Отвечай кратко и вежливо на отзыв покупателя…»);
    - роль `user` — текст отзыва (и при v2 — контекст похожих отзывов).
- Точные имена полей (`modelUri`, `messages`, `completionOptions`) уточнять в [документации Yandex GPT](https://cloud.yandex.ru/docs/yandexgpt/text-generation/api-ref/TextGeneration/completion) или в примерах API.

### 2.4 Формат ответа

- В ответе — сгенерированный текст (например, в поле `result.alternatives[0].message.text` или аналог). Извлечь строку и сохранить в локальную модель отзыва / очередь модерации.
- При ошибке — код и сообщение; не логировать ключ и полные ответы с персональными данными.

### 2.5 Для приложения

- Один вызов на отзыв при «Обработать» (или при фоновой обработке). Таймаут 30–60 с, retry при 5xx/429. Учёт лимитов Yandex (quota).

## 3. Yandex Embeddings API (для RAG — вторая итерация)

- **Назначение (v2):** получать вектор представления текста отзыва для поиска похожих в локальном архиве.
- **Эндпоинт:** из документации Yandex (например, отдельный endpoint для embeddings в Yandex Cloud).
- **Запрос:** текст(ы), модель эмбеддингов. **Ответ:** массив векторов. Хранить векторы локально (Room или файл) вместе с текстом отзыва; поиск похожих — косинусное сходство в приложении.
- В **MVP (v1)** эмбеддинги не используем; документ пригодится при добавлении RAG.

---

## 4. Обработка ошибок в приложении

- **401/403** — неверный ключ или нет доступа к каталогу: «Проверьте API-ключ и folder_id в настройках».
- **429** — превышена квота: повтор с задержкой, сообщение пользователю при постоянной перегрузке.
- **5xx / таймаут** — ограниченный retry, сообщение пользователю.
- Секреты не логировать.

---

## 5. Примеры для Retrofit (концептуально)

- **Базовый URL:** константа (BuildConfig или настройки), например `https://llm.api.cloud.yandex.net`.
- **Interceptor:** подставлять `Authorization: Api-Key <key>` и `x-folder-id: <folder_id>` из безопасного хранилища.
- **Сервис (псевдокод):**
  - `POST(".../completion") fun complete(@Body body: YandexCompletionRequest): Response<YandexCompletionResponse>`
- DTO описать по актуальному API (поля запроса/ответа могут отличаться).
