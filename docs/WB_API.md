# Wildberries API — вызовы с телефона (Фаза 0.2)

**Назначение:** приложение без бэкенда обращается к Wildberries API напрямую с устройства для получения отзывов и отправки ответов.  
**Официальная документация:** [dev.wildberries.ru — Отзывы и Вопросы](https://dev.wildberries.ru/ru/openapi/user-communication).

---

## 1. Базовый URL и аутентификация

- **Базовый URL API (отзывы):**  
  `https://feedbacks-api.wildberries.ru`  
  *(актуальный хост уточнять в документации WB; возможны варианты вроде `https://feedbacks-api.wb.ru` или иной поддомен — при реализации проверить по актуальной документации.)*

- **Аутентификация:** JWT-токен продавца в заголовке **`Authorization: Bearer <token>`**. Токен создаётся в ЛК WB (Профиль → Доступ к API) с правами «Отзывы и Вопросы». Сервер возвращает 401 с текстом "empty Authorization header", если заголовок отсутствует.
  Токен создаётся в личном кабинете Wildberries: **Профиль → Доступ к API** с правами на категорию **«Отзывы и Вопросы»** (и при необходимости «Контент»). Токен хранить только на устройстве в зашифрованном виде.

- **Лимиты:**  
  - 3 запроса в секунду;  
  - burst: 6 запросов за 333 мс.  
  В приложении нужна осторожная частота вызовов (throttling/retry с backoff при 429).

## 2. Эндпоинты, нужные приложению

### 2.1 Получение списка отзывов

- **Метод и путь:** `GET /api/v1/feedbacks`
- **Назначение:** получить список отзывов для сохранения в Room и отображения в приложении.
- **Параметры запроса (типичные):**
  - `take` — количество записей (например 50);
  - `skip` — смещение для пагинации;
  - `isAnswered` — фильтр по наличию ответа (true/false), опционально.
- **Ответ:** объект вида `{ "data": { "feedbacks": [ ... ] } }`, а не массив напрямую. Типичные поля (названия уточнять по API):
  - `id` — идентификатор отзыва;
  - `productDetails` / `nmId` / артикул;
  - `text` — текст отзыва;
  - `productValuation` / `rating` — оценка (например 1–5);
  - `createdDate` — дата;
  - `state` — статус (новый, ответ дан и т.д.);
  - при наличии: ответ продавца, имя автора и т.п.
- **Для приложения:** парсить JSON, маппить в локальные модели, сохранять в Room; при повторной загрузке обновлять по `id`.

### 2.2 Отправка ответа на отзыв

- **Метод и путь:** `POST /api/v1/feedbacks/answer` или `PATCH /api/v1/feedbacks/answer` (точный метод и путь — по актуальной документации WB).
- **Назначение:** отправить текст ответа продавца на отзыв (после одобрения или редактирования в приложении).
- **Тело запроса (типичное):** JSON, например:
  - `id` или `feedbackId` — id отзыва;
  - `text` — текст ответа.
- **Ответ:** успех/ошибка; при успехе — обновить локальный статус отзыва в Room (например «ответ отправлен»).
- **Для приложения:** вызывать после действия пользователя «Одобрить» или «Редактировать»; повтор при сетевой ошибке (с лимитами WB).

### 2.3 (Опционально) Количество неотвеченных отзывов

- **Метод и путь:** `GET /api/v1/feedbacks/count-unanswered` (или аналог из документации).
- **Назначение:** показать бейдж «N неотвеченных» без полной выгрузки списка.
- Реализовать по необходимости.

### 2.4 (Опционально) Архив отзывов

- **Метод и путь:** `GET /api/v1/feedbacks/archive`
- Для будущего RAG: выгрузка архива для построения локального хранилища похожих отзывов. В MVP не обязателен.

## 3. Обработка ошибок (в приложении)

- **401** — неверный/истёкший токен: показать пользователю сообщение «Проверьте токен WB в настройках».
- **429** — превышен лимит: повтор запроса с задержкой (exponential backoff), не чаще 3 запросов в секунду.
- **5xx / таймаут** — повтор с ограничением количества попыток, сообщение пользователю при неудаче.
- Не логировать токен и ответы с персональными данными в открытом виде.

## 4. Примеры для Retrofit (концептуально)

- **Базовый URL:** константа в приложении (BuildConfig или настройки), например `https://feedbacks-api.wildberries.ru`.
- **Interceptor:** добавлять заголовок с токеном из безопасного хранилища (EncryptedSharedPreferences/DataStore). Токен в запрос подставлять только в interceptor, не хранить в логах.
- **Интерфейс сервиса (псевдокод):**
  - `GET("api/v1/feedbacks") fun getFeedbacks(@Query("take") take: Int, @Query("skip") skip: Int): Response<List<FeedbackDto>>`
  - `POST("api/v1/feedbacks/answer") fun sendAnswer(@Body body: AnswerRequest): Response<Unit>`
- DTO описать по фактическому формату ответов WB (поля могут отличаться — при реализации сверить с актуальным API).
